# 万物有戏：Backend MVP 质量评分表

目标是用可复现的数据回答三个面试问题：产品是否真的能生成、是否好玩、成本是否可控。开发阶段先评估 10 张已获授权的照片，作品集公开前扩展到 50 张。

## 评估口径

- 生成成功率 = 成功进入案件的照片数 ÷ 总照片数
- 热点可用率 = 三个热点均落在对应物品上的成功案件数 ÷ 成功案件数
- 卡片降级率 = `CARD_FALLBACK` 案件数 ÷ 成功案件数
- 修复率 = 调用过一次 DeepSeek 修复的成功案件数 ÷ 成功案件数
- P50 / P95 时延 = 从创建任务到案件可进入的中位数 / 95 分位数
- 单局平均成本 = 成功案件的模型总成本 ÷ 成功案件数
- 首答正确率只用于观察难度，不作为生成质量的单一结论

建议 MVP 门槛：生成成功率 ≥ 80%，热点可用率 ≥ 70%，P95 ≤ 45 秒，单局平均成本在预设预算内，且 0 次答案提前泄露、0 张照片超过保留期限。
