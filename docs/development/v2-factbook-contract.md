# V2 Factbook 接口约定

本文固定 Phase 2 提供给后续“有限自由审问”和最终指认流程的数据边界。

## 案件事实簿

- 每个案件恰好包含三件唯一物证、三名唯一嫌疑人和三份公开证词。
- 每名嫌疑人拥有一份公开初始证词、一项私密行动、一组允许在审问中使用的事实 ID，以及一件与其绑定的物证。
- 公开 claim 只暴露证词文本；其 `factRefs` 与 `evidenceRefs` 仅保存在私有事实簿中。
- 私有事实簿是 `liarSuspectId`、唯一 contradiction 与完整 truth 的唯一来源。

## 浏览器与审问边界

- 案件完成前，浏览器不得收到说谎者、私密行动、允许事实 ID、矛盾解释、动机、证据链或完整真相。
- Phase 3 审问只能读取当前所选嫌疑人的允许事实子集；NPC 不得引用其他嫌疑人的私密事实，也不得创造新证据。
- 独立语义 judge 只能依据公开证词与物证判断矛盾是否唯一，不读取嫌疑人的私密行动或允许事实 ID。

## 后续提交接口

Phase 4 最终指认固定接收：

```ts
{
  accusedSuspectId: string;
  supportingEvidenceId: string;
}
```

服务端负责验证嫌疑人、物证和唯一矛盾的对应关系；客户端答案不作为可信真相来源。

## 兼容性

- V1 私有案件与玩家视图继续可读。
- V2 仍存放在既有 `jsonb` 字段中，不需要数据库迁移。
- 客户端持久状态使用 version 2；旧的瞬时探索状态被安全丢弃并从首页重新开始。
